Downstream Data Management Tool

The Downstream Data Utility is a new feature designed for Ford's internal Over-the-Air (OTA) updates management system. This feature enables engineers to edit or roll back software packages after deployment. Our goal was to design an intuitive interface that simplified complex rollback processes, supported batch operations, and provided clear visibility into system outcomes.

The Challenge

Engineers had no straightforward way to roll back software deployments.

Before this feature was introduced, engineers had no straightforward way to roll back software packages that had already been deployed to vehicles. This limitation created significant operational challenges across Ford's OTA infrastructure.

Manual Intervention

Rollback actions required manual intervention and multiple system steps with no unified interface.

Lack of Visibility

Engineers lacked clear visibility into which rollbacks succeeded or failed, leading to uncertainty.

Batch Operations

Managing multiple software packages at once was time-consuming and error-prone without batch tooling.

Platform Integration

The new feature needed to fit seamlessly into Ford's existing OTA platform without disrupting workflows.

Research & Discovery

Stakeholder interviews and collaborative brainstorming to map the problem.

I began by conducting stakeholder interviews and collaborative brainstorming sessions with engineers, product owners, and fellow designers to understand how rollback processes currently worked, what success and failure scenarios looked like, and how we could make the data easily accessible for the engineers.

Simplify Workflows

Reduce the number of steps required and enable batch rollbacks to improve operational efficiency.

Design Consistency

Use Ford's established internal design system to seamlessly blend with the rest of the OTA site.

Clear Feedback

Show rollback success or failure with clear status communication and steps to fix errors.

Final Design

A critical tool that set a new internal standard for usability.

The new Downstream Data Utility feature significantly improved the rollback process for Ford's internal OTA engineers. The feature became a critical tool within the OTA site, streamlining data management workflows and setting a new internal standard for usability in complex engineering systems.

Reduced Rollback Time

Multi-package batch actions dramatically reduced time required for rollback operations.

Increased Transparency

Clear success/failure feedback increased engineer confidence in system reliability and data accuracy.
